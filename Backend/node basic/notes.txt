HTTP status code

Status Code	Meaning	When to Use
400	Bad Request	Invalid input or missing required fields
401	Unauthorized	User is not logged in or token is missing/invalid
403	Forbidden	User is logged in but doesn't have permission
404	Not Found	Resource doesn't exist
405	Method Not Allowed	Wrong HTTP method (less common in Express by default)
409	Conflict	Duplicate resource (e.g., email already exists)
422	Unprocessable Entity	Validation failed (often used with schema validation)
429	Too Many Requests	Rate limit exceeded

500 internal server error
505 service unavailable
